# Contributing To SqliteSyncPro

Thanks for your interest in improving SqliteSyncPro.

## Best Ways To Help

- Report reproducible bugs in sync behavior, authentication, or setup.
- Suggest enhancements for offline-first application scenarios.
- Improve tests, examples, documentation, or setup instructions.
- Submit focused pull requests with clear reasoning and validation.

## Before You Open An Issue

Please:

1. Search existing issues first.
2. Use the issue form that best matches your request.
3. Include enough technical context to make the report actionable.

## Bug Reports

Helpful bug reports usually include:

- Library version or commit
- Operating system and Qt version
- Local database shape or simplified schema
- Remote stack details such as PostgreSQL, PostgREST, or Supabase
- Steps to reproduce
- Expected result
- Actual result
- Logs, payload samples, or screenshots when safe to share

If the issue involves encryption, authentication, or threading, please call that out early.

## Enhancement Requests

Enhancement requests are most useful when they explain:

- The application scenario you are trying to support
- Current limitations or pain points
- The proposed behavior or API change
- Tradeoffs, compatibility concerns, or alternatives considered

## Pull Requests

Please keep pull requests:

- Focused on one change set
- Small enough to review comfortably
- Consistent with the existing architecture and naming
- Backed by tests or clear manual validation notes when possible

When relevant, include:

- Test updates
- Documentation updates
- Notes about compatibility or migration impact

## Tests And Examples

If you change sync behavior, serialization, authentication, or threading, please update tests when practical. Improvements to the example app and admin tooling are also welcome.

## Collaboration

Please keep feedback respectful, precise, and constructive. Good technical collaboration saves time for everyone.
